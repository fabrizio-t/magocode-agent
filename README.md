# MagoCode Agent

Outbound agent for MagoCode-managed machines and self-hosted Linux nodes.

## What It Does

The agent runs on a user's machine or VPS and opens an outbound WebSocket connection to MagoCode. Through that connection MagoCode can:

- run shell commands
- open interactive PTY sessions
- tail log files
- read/write files in approved task paths
- report basic runtime metadata and capabilities

The agent does not require inbound SSH or open firewall ports.

## Security Model

This agent can execute commands on the machine where it runs. Only install it on machines you intend MagoCode to control.

The agent opens an outbound WebSocket connection to MagoCode. It does not require inbound SSH, inbound HTTP, or open firewall ports.

Agent authentication uses a machine-scoped API key generated by MagoCode. Store that key like a password. If a key is exposed, delete or revoke the agent in MagoCode and reinstall with a new key.

Current capabilities:

- `tail`
- `exec`
- `writeFile`
- `readFile`
- `fileExists`
- `pty`

Path-scoped file access is limited to:

- `/home/magocode/**`
- `/tmp/magocode-*`

## Configuration

By default the agent reads:

```text
/home/magocode/.magocode.json
```

Override with:

```bash
MAGOCODE_AGENT_CONFIG=/path/to/config.json npm start
```

Example config:

```json
{
  "instance": "https://api.magocode.com",
  "apiKey": "agent-vps-example:secret"
}
```

`instance` may be `https://`, `http://`, `wss://`, or `ws://`. The agent connects to `/agent`.

## Install

The first standalone install flow uses a git checkout on the VPS and a systemd user service:

```bash
curl -fsSL https://raw.githubusercontent.com/fabrizio-t/magocode-agent/v0.2.1/install.sh | sudo bash -s -- --api-key 'agent-sub:secret'
```

See [docs/install.md](docs/install.md).

## Development

```bash
npm install
npm test
npm start
```

For local testing:

```bash
MAGOCODE_AGENT_CONFIG=./magocode-agent.local.json npm start
```
